The gap

AI 도입은 툴 구매보다
기준 설계가 먼저입니다.

누가 어떤 업무에 AI를 써도 되는지, 어떤 정보는 넣으면 안 되는지, 결과물을 누가 검토해야 하는지 정하지 않으면 도입은 빠르게 번지지만 책임은 남습니다.

For employees

“이 자료를 AI에 넣어도 되나?”

실무자는 편하게 쓰고 싶지만 회사자료, 고객정보, 외부 제출물의 기준이 없으면 매번 눈치와 감으로 판단합니다.

For leaders

“어디부터 도입해야 효과가 나나?”

대표와 팀장은 AI를 도입하고 싶어도 효과, 위험, 예산, 보안 설득자료가 없어 실행 우선순위를 정하기 어렵습니다.

For security

“사용은 늘었는데 기록이 없다.”

보안·감사 담당자는 입력 금지 정보, 사용 로그, 승인 절차, 책임 기준이 없어 사후 대응이 어려워집니다.

AI 사용정책 초안

Who it helps

같은 진단, 다른 답을 줍니다.

처음부터 한 고객만 보지 않고, 직장인·대표/도입 담당자·보안/감사 담당자의 반응을 나눠 측정할 수 있게 설계했습니다.

직장인·실무자

AI를 업무에 쓰고 있지만 어디까지 괜찮은지 헷갈리는 사용자.

  • 업무별 사용 가능 범위
  • 외부 제출 전 검토 기준
  • 회사자료 입력 주의사항

대표·도입 담당자

AI를 도입해야 하지만 보안·효과·예산 설득자료가 필요한 의사결정자.

  • 도입 준비도 요약
  • 파일럿 우선 업무 추천
  • 내부 승인용 리포트

보안·감사 담당자

직원들의 AI 사용을 막기보다 안전하게 관리해야 하는 책임자.

  • 입력 금지 정보 기준
  • 사용 로그·승인 체계
  • 사고·감사 대응 근거
Pilot flow

3분 사전진단으로 시작해
2주 파일럿 범위까지 좁힙니다.

초기 단계에서는 완성품을 과장하지 않고, 샘플 리포트와 파일럿 안내를 통해 실제 반응과 페이포인트를 확인합니다.

01

현재 상황 선택

실무 사용 중인지, 대표가 도입을 검토 중인지, 보안정책을 세워야 하는지 먼저 구분합니다.

02

위험 업무 확인

보고서, 고객응대, 개인정보 문서, 외부 제출물 등 실제 AI 사용 업무를 기준으로 위험도를 나눕니다.

03

리포트 샘플 제공

AI 사용 위험 지도, 도입 준비도, 정책 공백, 파일럿 우선순위를 요약한 샘플을 제공합니다.

04

파일럿 안내

반응이 있는 조직에는 1개 팀 또는 1개 업무부터 검증하는 고정 범위 파일럿을 제안합니다.

AgentProof는 현재 초기 고객검증 단계입니다. 실제 내부자료나 민감정보는 제출하지 않아도 됩니다.
