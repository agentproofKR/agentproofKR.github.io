# 05. 시각 검수 기준 — AgentProof V4

## 1. 목적

Codex 구현물을 최종 승인된 PC·모바일 캡처와 비교한다. 기존 HTML은 상호작용 참고용이며, 카피와 최종 시각 구성은 `reference/visual-baseline`이 우선한다.

## 2. 필수 기준 파일

- `reference/agentproof_v4_target_desktop.png`
- `reference/agentproof_v4_target_mobile.png`
- `reference/visual-baseline/README.md`
- `reference/visual-baseline/capture-manifest.json`

## 3. 검수 절차

1. 1440px 폭에서 full-page 캡처를 생성한다.
2. 390px 폭에서 full-page 캡처를 생성한다.
3. 기준 이미지와 pixel diff를 생성한다.
4. 구간별 캡처를 나란히 놓고 H1, 앱 화면, 역할, 판단 흐름, 다크 섹션을 사람 눈으로 검수한다.
5. 차이가 있으면 CSS를 수정하되 카피를 바꾸어 맞추지 않는다.
6. PC·모바일 모두 통과한 뒤 1280, 1024, 768, 360, 320px 회귀 검수를 실행한다.

## 4. 시각 승인 기준

- 기준 이미지와 동일한 정보 구조와 섹션 순서
- H1 줄바꿈 동일
- PC 최대 콘텐츠 폭과 좌우 여백 동일한 인상
- 모바일 CTA 높이와 좌우 여백 동일
- 진단 화면이 샘플 제품 화면으로 읽히며 텍스트가 무너지지 않음
- 밝은 섹션에서 다크 섹션으로 전환되는 지점과 대비 유지
- 폰트 렌더링 차이를 제외한 주요 박스 위치 오차 8px 이내
- CTA, 헤더, 카드의 radius와 border가 눈에 띄게 달라지지 않음

## 5. 제출 증거

PR 또는 완료 보고에 아래를 첨부한다.

- 새 PC full-page 캡처
- 새 모바일 full-page 캡처
- PC diff 이미지
- 모바일 diff 이미지
- Playwright visual test 결과
- 의도적으로 남긴 차이와 이유
