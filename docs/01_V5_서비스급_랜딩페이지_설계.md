# AgentProof V5.0 — 서비스급 랜딩페이지 설계

- 상태: 개발 착수 가능
- 기준 해상도: Desktop 1440px / Mobile 390px
- 핵심 목표: 초기 프로젝트 소개 페이지가 아니라, 실제 B2B SaaS 서비스가 운영 중인 것처럼 신뢰·제품 이해·상담 전환이 이어지는 랜딩페이지를 만든다.
- 시각 기준: `agentproof_v5_desktop_full.png`, `agentproof_v5_mobile_full.png`
- 제품 기준: `agentproof_mvp_dashboard_agentproof.png`

## 1. 최종 결정

AgentProof는 특정 구축사만을 위한 납품 검수 서비스로 보이지 않아야 한다. 랜딩페이지의 상위 포지셔닝은 다음 세 역할을 하나의 제품 경험 안에 묶는다.

1. 업무에 AI를 사용하는 실무자
2. AI 도입을 결정하는 대표·도입 담당자
3. 보안·정책·감사 담당자

최종 한 문장 정의:

> AgentProof는 사내 문서와 규정을 사용하는 업무용 AI의 답변 근거, 정확성, 정책·권한 위험을 검증하고 배포 승인과 운영 기록까지 한 화면에 정리합니다.

핵심 메시지:

> AI를 업무에 쓸 때, 무엇을 맡기고 어디까지 허용할지.

## 2. 서비스급으로 올리기 위해 바꾼 원칙

### 2.1 제품을 먼저 보여준다

추상적인 위험 설명이나 긴 문제 제기보다, 첫 화면 직후 실제 제품 대시보드를 크게 보여준다. 사용자는 설명을 읽기 전에 다음을 확인할 수 있어야 한다.

- 위험 점수
- 검증 통과율
- 배포 승인 상태
- 열린 이슈
- Agent 목록
- 답변과 근거 문서
- 정책·환각·최신성·권한 테스트
- 감사 리포트와 승인 요청

### 2.2 한 화면에 하나의 주장만 둔다

페이지 흐름을 네 개의 질문으로 단순화한다.

1. AgentProof가 무엇인가?
2. 실제 화면에서 무엇을 확인하는가?
3. 역할별로 어떤 결정을 돕는가?
4. 어떻게 파일럿을 시작하는가?

### 2.3 신뢰를 가짜 실적으로 채우지 않는다

확인되지 않은 고객 로고, 후기, 인증, 성과 숫자는 사용하지 않는다. 대신 다음으로 신뢰를 만든다.

- 실제 제품 수준의 대시보드
- 명확한 `SAMPLE DATA` 표시
- 구체적인 파일럿 결과물
- 상담 시 실제 내부 자료가 필요하지 않다는 안내
- 역할별 사용 가치와 검증 흐름

### 2.4 CTA는 하나의 전환 경로로 통일한다

모든 주요 CTA는 동일한 리드 모달로 연결한다.

- 헤더: `도입 진단 요청`
- 히어로: `우리 조직 적용 범위 확인`
- 프로세스: `파일럿 적용 범위 확인`
- 최종 CTA: `도입 진단 요청`

버튼 문구는 다르지만 제출 폼과 데이터 구조는 하나다. `placement` 값으로 클릭 위치를 구분한다.

## 3. 서비스 벤치마크에서 가져온 패턴

- Linear: 짧은 핵심 문장과 대형 제품 화면 중심의 서사
- Intercom: 명확한 주 CTA와 보조 데모 CTA
- Credal: 엔터프라이즈 Agent를 관리·통제하는 제품 정의
- Vanta: 위험을 발견하고 개선·증명하는 운영 흐름
- Glean: 실무 사용성과 엔터프라이즈 신뢰를 함께 표현

그대로 복제하지 않고 AgentProof에 맞게 다음 구조로 통합한다.

> 짧은 약속 → 제품 화면 → 역할별 가치 → 검증·승인 흐름 → 단일 상담 전환

참고 URL:

- https://linear.app/
- https://www.intercom.com/
- https://www.credal.ai/
- https://www.vanta.com/trust-management-platform
- https://www.glean.com/

CTA:

- Primary: `우리 조직 적용 범위 확인`

신뢰 보조 문구:

- 실제 내부 자료 없이 1차 진단

좌측 캡션:

> PRODUCT PREVIEW · Agent 운영 및 검증

우측 캡션:

> 화면 내 수치는 샘플 데이터입니다.

제품 탭:

1. `근거 기반 답변`
2. `자동 검증`
3. `승인과 감사 기록`

### 5.4 역할별 가치

제목:

> 세 팀이 같은 화면에서,  
> 다른 결정을 내립니다.

설명:

> AI 도입은 도구를 고르는 일이 아니라, 실무·경영·보안이 같은 근거로 허용 범위를 합의하는 일입니다.

역할:

- 실무자: 검토 가능한 답변
- 대표·도입 담당자: 근거가 남는 도입 결정
- 보안·정책 담당자: 운영 가능한 통제 기준

### 5.5 검증 흐름

```text
업무·문서·Agent
→ 실제 업무 질문
→ 정확성·근거·정책·권한
→ 사람 검토·배포 승인
→ 테스트·감사 리포트
```

### 5.6 다크 섹션

제목:

> AI를 막기보다,  
> 검증하고 승인해 운영합니다.

설명:

> 한 번의 체크리스트로 끝내지 않습니다. 실제 업무와 문서를 기준으로 테스트하고, 사람이 승인할 수 있는 근거와 이후 운영 기록을 남깁니다.

3단계:

1. 적용 범위 정의
2. 검증 시나리오 실행
3. 승인·운영 기준 확정

### 5.7 파일럿 결과물

- AI 적용 범위 지도
- 검증 시나리오와 결과
- 승인·운영 리포트

### 5.8 최종 CTA

> 우리 조직의 첫 AI 사용 기준부터 정리해보세요.

## 6. 제품 대시보드 설계

### 6.1 브랜드 수정

원본 MVP의 `Agent Assurance`는 모든 화면과 이미지에서 `AgentProof`로 교체한다. 상단 사용자 실명은 사용하지 않고 다음으로 통일한다.

- 이름: `관리자`
- 역할: `보안 담당자`

### 6.2 데스크톱

데스크톱에서는 전체 대시보드 구조를 보여준다.

- 네이비 상단 바
- 좌측 내비게이션
- KPI 4개
- Agent 목록
- 질문·답변과 근거 문서
- 최근 검증 결과
- 감사 리포트 생성
- 배포 승인 요청

제품 프레임은 최대폭 1,200px이며, 원본 이미지의 약 2:1 비율을 유지한다. 프레임 내부 이미지에 `cover`를 사용하지 않는다. 잘림 없이 `width:100%; height:auto`로 렌더링한다.

### 6.3 모바일

모바일은 데스크톱 대시보드의 단순 축소가 아니다. 390px 전용 구성으로 다시 배열한다.

1. AgentProof 상단 바 + SAMPLE DATA
2. 화면 제목
3. KPI 2×2
4. 근거가 있는 답변
5. 근거 문서
6. 최근 검증 결과
7. 감사 리포트 / 배포 승인

모바일에서는 좌측 내비게이션과 Agent 목록을 숨긴다. 글자 크기는 7px 미만으로 내려가지 않으며, 페이지 가로 스크롤은 절대 허용하지 않는다.

## 10. 시각 검수 기준

### 데스크톱 1440px

- H1은 두 줄
- 제품 화면 좌우 잘림 없음
- 제품 화면 바로 아래 탭 3개
- 역할 카드 3열
- 다크 섹션 좌측 설명 / 우측 결과물 패널
- 전체 높이 기준 약 4,050px

### 모바일 390px

- H1은 3줄 또는 4줄 이내
- 상단 CTA가 헤더 안에 유지
- 전용 모바일 대시보드 사용
- 역할 카드 1열
- 판단 흐름 1열
- 다크 섹션 결과물 패널은 본문 다음에 배치
- 전체 높이 기준 약 5,250px

### 즉시 실패

- Agent Assurance 텍스트 잔존
- 실명 노출
- SAMPLE DATA 미표시
- 모바일 대시보드 전체 축소
- 가로 스크롤
- 제품 화면 잘림 또는 비율 왜곡
- 가짜 고객 로고·후기·인증 추가
- 주요 CTA가 서로 다른 폼으로 연결
