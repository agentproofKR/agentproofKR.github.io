# 질문 은행과 점수 규칙

## 버전

- Survey version: `2026-06-21`
- Scoring version: `2026-06-21`
- 구현 파일: `lib/survey/questions.ts`, `lib/survey/scoring.ts`

## 질문 수

- 공통 질문: 6개
- 실무자: 공통 6개 + P07-P24 = 24개
- 대표·도입 담당자: 공통 6개 + L07-L25 = 25개
- 보안·정책 담당자: 공통 6개 + S07-S26 = 26개

## 점수 규칙

- 기본 성숙도: 없음 0, 계획 중 1, 일부 적용 2, 대부분 적용 3, 정착·정기점검 4.
- 모름: 0점이며 정보 공백 flag를 만든다.
- 해당 없음과 응답하지 않음: denominator에서 제외한다.
- 세그먼트 질문은 점수에서 제외한다.
- multi-select 점수 문항은 선택 항목 중 가장 낮은 점수를 적용한다.
- 위험 행동 문항은 reverse scoring을 적용한다.
- 각 차원은 0-100으로 normalize한다.

## 결과 band

- 0-39: 기준 정립 필요
- 40-59: 제한적 실험 적합
- 60-79: 통제 기반 확대 준비
- 80-100: 운영 고도화 단계

## Critical warning override

아래 항목이 있으면 effective band는 `기준 정립 필요`로 내려간다.

- 소비자용 AI에 개인정보 또는 기밀정보 입력 가능
- 외부 제출물이 사람 검토 없이 사용 가능
- 승인된 AI 목록 또는 사용정책 없음
- 행동 수행 Agent에 사람 승인 절차 없음
- 질문·답변·행동 로그 없음
- 공급자의 모델 학습 또는 국외 이전 상태 불명
